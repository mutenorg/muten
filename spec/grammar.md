# `.screen` language rules

> The authoritative source of the **syntax**. The **vocabulary** (primitives, props, tokens,
> keywords) lives in [`src/manifest.js`](../src/manifest.js) — the other half of the rules.
> From these two things derive: the parser (`src/parse.js`), the validator (`src/validate.js`),
> the highlight (generated by `tools/gen-grammar.mjs`), the linter and the autocomplete.
> **Changing the language = changing these rules + the manifest, and regenerating.**

## 1. Lexical (tokens)

```
comment    = "#" .* (to end of line)            # ignored
string     = '"' .* '"'
ref        = "@" ident                          # state reference:  @users
param      = "$" ident                          # part-param reference: $title
number     = "-"? digit+ ( "." digit+ )?
ident      = (letter | "_") (letter | digit | "_")*
operators  = "->"  "<-"  "=>"  "=="  "|"
punctuation = { } ( ) [ ] , : = < > .
```

## 2. File structure

A `.screen` is a sequence of declarations, in any order:

```
file        = declaration* ;
declaration = screen | entity | state | mock | action | part | root-node ;
```

- A **page** (`pages/<route>/<route>.screen`) usually has `screen` + `state`/`entity`/`action`/`mock` + a **root node** (the tree).
- A **part file** (`parts/<n>.screen` or `src/parts/`) has `part` (+ its own `entity`/`state`/`mock`, which are *hoisted* when the part is used).

## 3. Declarations

```
screen   = "screen" ident ;

entity   = "entity" ident "{" field* "}" ;
field    = ident fieldType ;
fieldType= ident ( "|" ident )* ;          # one ident = type; many = enum.  id:uuid is implicit.

state    = "state" "{" stateDecl* "}" ;
stateDecl= ident "=" init ":" type ;
init     = "query" ident | "{" "}" | string | number | ident ;
type     = ident ( "<" ident ">" )? ;       # e.g. list<User>

mock     = "mock" "{" ( ident ":" value )* "}" ;
value    = string | number | "true" | "false" | "null" | ident | array | object ;
array    = "[" ( value ( "," value )* )? "]" ;
object   = "{" ( ident ":" value ( "," … )* )? "}" ;

action   = "action" ident "mutates" ident ( "," ident )* "<-" ident "{" statement* "}" ;
statement= ident "." method "(" args ")" ;  # method ∈ manifest.ACTION_OPS (push|remove|reset|set)

part     = "part" ident "(" ( param ( "," param )* )? ")" "{" node* "}" ;
param    = ident ":" type ;
```

## 4. Nodes (the UI tree)

```
node          = primitive | partInstance ;

primitive     = TYPE nodePart* block? ;           # TYPE ∈ manifest.PRIMITIVES
partInstance  = ident "(" ( namedArg ( "," … )* )? ")" ;   # ident is NOT a primitive
namedArg      = ident ":" argValue ;
argValue      = string | number | ref | param | ident ;

nodePart      = positional | modifier ;
positional    = string                            # -> the primitive's string-prop (Text "x" => value)
              | ref                               # positional @x => prop `data`
              | param                             # $x  (inside a part)
              | "->" ident ( "(" arg ")" )? ;     # row action:  -> deleteUser(row.id)
modifier      = "bind" ref                        # bind @search
              | "submit" ident                    # submit createUser
              | "where" "(" clause* ")"           # where(role == admin, name contains @q)
              | "columns" "(" ident* ")"          # columns(name, email)
              | "style" "(" token* ")" ;          # style(surface, padding.lg)   token ∈ manifest.TOKENS
block         = "{" node* "}" ;
```

**Disambiguation rule:** if `(` comes right after the `TYPE`, it's a **part instance**;
otherwise it's a **primitive** with its parts. Primitives never take `(args)` after the name.

## 5. From rules to IR

- Each `node` → `{ type, props, children?, loc }`; a part instance → `{ type, args, loc }`.
- The **positional string** is assigned to the prop given by `manifest.PRIMITIVES[TYPE].string`.
- `compose` inlines part instances (substituting `$param` with the args) **before** flattening:
  the part disappears and a tree of primitives remains.
- `validate` requires: known types/parts, existing `@refs` in state, valid tokens,
  **required props** (the manifest ones without `?`), and actions only mutate what's declared.

## 6. The ecosystem (one source, many tools)

```
            rules (this doc)  +  manifest (vocabulary)
                          │
   ┌──────────┬───────────┼───────────┬──────────────┐
 parse      validate   highlight    linter        autocomplete
(src)        (src)    (generated)   (analyze)        (analyze)
```

Everything reads the same thing. Adding a primitive, a token or a keyword = edit the manifest
(and its render in `compile.js`) and **regenerate** the highlight — the rest stays consistent on its own.
